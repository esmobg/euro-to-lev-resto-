# Инструкции за публикуване на калкулатора

## Опция 1: GitHub Pages (Безплатно и лесно) ⭐ РЕКОМЕНДИРАНО

### Стъпки:
1. Създайте GitHub акаунт на https://github.com (ако нямате)
2. Създайте нов repository (хранилище)
3. Качете файловете:
   - `index.html`
   - `styles.css`
   - `script.js`
   - `accessibility-test.html` (опционално)
4. Отидете в Settings → Pages
5. Изберете branch: `main` и folder: `/ (root)`
6. Вашият сайт ще бъде достъпен на: `https://вашето-потребителско-име.github.io/име-на-repository`

### Предимства:
- ✅ Безплатно
- ✅ Лесно за използване
- ✅ Автоматично обновяване при push
- ✅ HTTPS сертификат включен

---

## Опция 2: Netlify (Безплатно и много лесно) ⭐ РЕКОМЕНДИРАНО

### Стъпки:
1. Отидете на https://www.netlify.com
2. Регистрирайте се (безплатно)
3. Drag & Drop папката с файловете или свържете с GitHub
4. Сайтът е готов веднага!

### Предимства:
- ✅ Безплатно
- ✅ Много лесно (drag & drop)
- ✅ Автоматично HTTPS
- ✅ Може да свържете с GitHub за автоматично обновяване

---

## Опция 4: Firebase Hosting (Безплатно)

### Стъпки:
1. Инсталирайте Firebase CLI: `npm install -g firebase-tools`
2. Регистрирайте се на https://firebase.google.com
3. Създайте проект
4. В папката с файловете изпълнете:
   ```bash
   firebase login
   firebase init hosting
   firebase deploy
   ```

### Предимства:
- ✅ Безплатно
- ✅ Добър за по-сложни проекти
- ✅ Много функции

## Какво да качите:

### Задължителни файлове:
- ✅ `index.html` - основната страница
- ✅ `styles.css` - стиловете
- ✅ `script.js` - JavaScript логиката

### Опционални файлове:
- `accessibility-test.html` - тестовата страница
- `README.md` - документация
- `.gitignore` - ако използвате Git

---

## Бърз старт с Netlify (Най-лесно):

1. Отидете на: https://app.netlify.com/drop
2. Плъзнете папката с файловете в прозореца
3. Получете линк веднага!

---

## Бърз старт с GitHub Pages:

1. Създайте нов repository на GitHub
2. Качете файловете
3. Settings → Pages → Source: main branch
4. Готово!

---

## Важно:

- Всички файлове трябва да са в една папка
- Не променяйте имената на файловете
- Уверете се, че `index.html` е в корена на папката

---

## Поддръжка:

Ако имате проблеми, проверете:
- Всички файлове са качени
- Имената на файловете са правилни
- Няма грешки в конзолата на браузъра (F12)
